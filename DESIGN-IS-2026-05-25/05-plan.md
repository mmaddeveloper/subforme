# 05 — Implementation plan: subforme v1.6 redesign

Generated by `/make-plan` from the Dieter Rams audit at `02-scorecard.md` (17/30 → REDESIGN). Discovery details at the bottom of this file.

## Plan-wide ground rules (apply to every phase)

- **Repository**: `/root/subforme`. Single Jinja template. No build step. No CSS framework. Vanilla JS, hand-rolled CSS.
- **Files in scope**: `/root/subforme/index.html` (primary, 3086 lines) + `/root/subforme/preview.html` (mirror, 3090 lines — 4 extra sample-data lines). Both must stay JS-identical at the main `<script>` block.
- **Files OUT of scope**: `/root/subforme/bridge/bridge.py`, `/root/subforme/install.sh`, `/root/subforme/uninstall.sh`. Do NOT propose changes that depend on the bridge. The `/api/sub/online/{token}` + `/static/<file>` contracts are frozen.
- **Migration mechanism**: there is no `?v=2` query-flag option — PasarGuard installs ONE template at a time. Cutover = a tagged release. New release is `v1.6` (current is `v1.5`). The user re-runs `install.sh` after the release is published. There is no parallel deploy — pick a moment, ship.
- **Each phase is self-contained**: a fresh chat starting from any phase should succeed using only this file + the artifacts in `/root/subforme/DESIGN-IS-2026-05-25/`. Quote line ranges; do not say "see the audit."
- **Verification gate** between phases: `python3 /tmp/extract_js.py && node --check /tmp/index.main.js && node --check /tmp/preview.main.js && diff -q /tmp/index.main.js /tmp/preview.main.js`. If any check fails, fix before moving on.
- **Commit cadence**: one commit per phase, on `main` directly (no PRs — per user memory `feedback_subforme_workflow`). Push after each commit so a new chat starts from clean origin.
- **No new dependencies**: no NPM packages, no new external scripts. ECharts stays self-hosted via the bridge. qrcode-generator (already inlined at line 11) stays.
- **i18n discipline**: never rename a key in `I18N.fa`/`I18N.en`. Only edit values, only add new keys, only delete a key after grep confirms zero references in HTML + JS.
- **Anti-pattern guards** (reject in your own diff before committing):
  - Porting old structure under new styling — if 9 sections + 2 modals + 4 copy buttons still exist after Phase 4, the redesign hasn't happened.
  - Sneaking ornaments back under new names — "subtle backdrop", "light shimmer", "tasteful glow" are the discarded layer with a new label.
  - Adding 2026 trend markers (glassmorphism, aurora, gradient mesh, AI-dashboard glow).
  - Renaming I18N keys or breaking the label↔handler invariant.

## Phase 1 — Design tokens + light/dark + reset

**Goal**: Rebuild the design-token foundation. Light mode is the daylight default; midnight stays as a user-selectable persisted theme. Cap the type/spacing/color scales.

**Exact line ranges to edit**:
- `<style>` block opens at `index.html:13`; `:root` at `106–110`; theme blocks at `17–37` (midnight), `39–60` (gold), `61–82` (emerald), `83–104` (crimson).
- Hardcoded theme attribute: `<html lang="fa" dir="rtl" data-theme="midnight">` at `index.html:2`.

**What to implement**:

1. **Drop the hardcoded theme** at `index.html:2`:
   ```html
   <html lang="fa" dir="rtl">
   ```
   Theme resolution moves to JS `applyTheme()` (existing) — must default to `light` when no `localStorage.getItem('pc-theme')` is set AND `matchMedia('(prefers-color-scheme: dark)')` is `false`.

2. **Add the new token system** as the first rule inside `<style>`. Replace the existing `:root` block at `106–110`:
   ```css
   :root {
     /* Type scale — cap 6 sizes */
     --t-xs: 12px;
     --t-sm: 14px;
     --t-md: 16px;
     --t-lg: 20px;
     --t-xl: 28px;
     --t-xxl: 40px;

     /* Spacing scale — cap 6 steps (4-base) */
     --s-1: 4px;
     --s-2: 8px;
     --s-3: 12px;
     --s-4: 16px;
     --s-5: 24px;
     --s-6: 32px;

     /* Radii */
     --r-sm: 6px;
     --r-md: 10px;
     --r-lg: 16px;

     /* Functional colors (light default) */
     --bg:        #fafafa;
     --surface:   #ffffff;
     --surface-2: #f5f5f5;
     --line:      #e4e4e7;
     --ink:       #18181b;
     --ink-2:     #3f3f46;
     --ink-3:     #52525b;       /* contrast 7.5:1 on --surface — passes AA */
     --ink-mute:  #71717a;
     --ok:        #15803d;
     --warn:      #b45309;
     --err:       #b91c1c;
     --accent:    #6d28d9;       /* indigo — desaturated for light bg */
     --accent-ink: #ffffff;      /* text on accent */
   }
   ```

3. **Add the dark theme block** that wins when `data-theme="midnight"` is set OR (no localStorage value AND `prefers-color-scheme: dark`):
   ```css
   :root[data-theme="midnight"],
   :root:not([data-theme]) {
     @media (prefers-color-scheme: dark) {
       /* this nested @media is fine in modern browsers; if it breaks
          older targets, hoist it out and gate with a JS media query
          listener inside applyTheme() */
     }
   }
   :root[data-theme="midnight"] {
     --bg:        #0a0a0f;
     --surface:   #14141c;
     --surface-2: #1c1c26;
     --line:      #2a2a38;
     --ink:       #f5f5f7;
     --ink-2:     #c4c4cf;
     --ink-3:     #9b9bab;       /* darkened from #6a6a78 → contrast ≥ 4.5:1 */
     --ink-mute:  #6a6a78;
     --ok:        #4ade80;
     --warn:      #fbbf24;
     --err:       #f87171;
     --accent:    #a78bfa;       /* lifted from #7c5cff for dark bg */
     --accent-ink: #0a0a0f;
   }
   ```
   The legacy 4-variant theme (midnight/gold/emerald/crimson) is collapsed to **2 themes**: `light` (default) + `midnight`. Gold/emerald/crimson are deleted in this phase — they were never load-bearing and they bloat the token surface. (Their accent hex values were preserved into `00-scope.md` for the record.)

4. **Add a reduced-motion guard at the top of the style block** (after `:root`):
   ```css
   @media (prefers-reduced-motion: reduce) {
     *, *::before, *::after {
       animation-duration: 0.01ms !important;
       animation-iteration-count: 1 !important;
       transition-duration: 0.01ms !important;
       scroll-behavior: auto !important;
     }
   }
   ```

5. **Update `applyTheme()`** (find it via grep — currently sets `data-theme`). New behavior:
   - Read `THEME` from localStorage; if absent, do NOT default to `'midnight'` — leave attribute UNSET so the OS media query wins.
   - Theme picker UI only offers `light` and `midnight` (delete gold/emerald/crimson from `THEMES` array; preserve i18n key `pickTheme`).

**Verification checklist for Phase 1**:
- [ ] `grep 'data-theme="midnight"' index.html` returns ZERO HTML hits (JS-set is OK; check the `<html>` line itself).
- [ ] In a browser at OS-light mode, default render shows light surfaces.
- [ ] In a browser at OS-dark mode (no localStorage), default render shows midnight surfaces.
- [ ] After picking midnight explicitly, the choice persists across reload (already-working pathway).

- [ ] `node --check` passes.

**Anti-pattern guards**:
- Don't keep the legacy `--accent-2`, `--accent-3`, `--aurora-1/2/3` tokens — they were tied to the removed visual layer. Grep them out.

## Phase 4 — Single canonical copy/QR surface

**Goal**: Replace 4 duplicated copy/QR affordances with one canonical pattern.

**Current locations of duplicates** (from discovery):
- Header CTA copy button at `index.html:1277` (`copySub()`)
- Header CTA "Show QR" button at `index.html:1283` (`showSubQR()`)
- Per-config copy `data-a="copy"` at `index.html:2224`
- Per-config QR `data-a="qr"` at `index.html:2229`
- Per-device copy + per-device QR inside `renderDevicesList()` template
- QR modal's standalone "Copy" button at `index.html:1393` (`copyFromModal()`)
- QR modal's standalone "Base64" button at `index.html:1394` (`copyB64FromModal()`)

**Target pattern** — one canonical place where a user copies a config/sub URL:

1. **Header subscription row** keeps ONE primary action: "Copy" (the subscription URL). The "Show QR" button moves to an overflow `⋯` menu next to it. The QR modal becomes view-only — no buttons inside it.

2. **Per-config row** is the same pattern: one visible "Copy" + a `⋯` that opens an action sheet with two items: "Show QR" + "Copy as Base64".

3. **Per-device row** has NO copy/QR affordance (devices aren't a config — they're status). Remove the per-device copy/QR buttons.

4. **Action sheet implementation**: Use a `<dialog>` element (already supported by all modern browsers; no polyfill needed) positioned bottom-sheet style on mobile, center on desktop. Inside: two big tap targets ("Show QR" → opens QR modal in view-only mode; "Copy as Base64" → writes `btoa(value)` to clipboard + toast).

5. **Delete from the QR modal** (`index.html:1377–1397`):
   - The "Copy" button at `1393` (`copyFromModal()` function at `2957–2959` — delete the function too if no other caller).
   - The "Base64" button at `1394` (`copyB64FromModal()` at `2960–2967` — function still needed; move it to the action sheet handler).
   The modal now contains: header (title, close) + body (label, sub, qr-frame). That's it.

6. **HTML structure for the canonical pattern**:
   ```html
   <div class="config-row">
     <div class="config-meta">…name + tag…</div>
     <button class="btn-copy" data-action="copy" data-target="{value}">
       <span data-i18n="copy">Copy</span>
     </button>
     <button class="btn-more" data-action="more" data-target="{value}" aria-label="More options">⋯</button>
   </div>
   ```
   One delegated click handler on the configs container distinguishes by `data-action`.

7. **i18n additions**: `more` (added in Phase 3), `showQR` (already exists as `scanQR`? — check, reuse if so), `copyB64` (already exists).

**Verification checklist for Phase 4**:

## Phase 5 — Missing states (error, disabled, loading)

**Goal**: Author the missing CSS states identified in the audit (`02-scorecard.md` #8 = 1/3).

**What to implement**:

1. **`.error` state** for failed network calls:
   ```css
   .net-error {
     border-radius: var(--r-md);
     border: 1px solid var(--err);
     background: color-mix(in srgb, var(--err) 8%, var(--surface));
     color: var(--err);
     padding: var(--s-3);
     font-size: var(--t-sm);
   }
   .net-error .retry {
     /* a button that re-runs the failed fetch */
   }
   ```
   Used in `renderDevices()`, `renderUsageStats()`, and any `fetchOnlineDevices()` catch block. When the bridge is unreachable, the device list shows `.net-error` with a "Retry" button + a one-line plain explanation. Toast is the stopgap; this is the state.

2. **`[aria-disabled="true"]`** styling for buttons during async actions:
   ```css
   button[aria-disabled="true"] {
     opacity: 0.5;
     cursor: not-allowed;
     pointer-events: none;
   }
   ```
   In the canonical "Copy" handler, set `aria-disabled="true"` on the button for the brief await + restore it in `.finally()`. Prevents double-tap from firing the clipboard write twice.

3. **Skeleton CSS** for the device list + usage chart:
   ```css
   .skeleton {
     background: linear-gradient(
       90deg,
       var(--surface-2) 0%,
       var(--line) 50%,
       var(--surface-2) 100%
     );
     background-size: 200% 100%;
     border-radius: var(--r-md);
     min-height: 1em;
   }
   @media (prefers-reduced-motion: no-preference) {
     .skeleton {
       animation: skeleton-shimmer 1.4s ease-in-out infinite;
     }
   }
   @keyframes skeleton-shimmer {
     0%   { background-position: 200% 0; }
     100% { background-position: -200% 0; }
   }
   ```
   `renderDevices()` paints `.skeleton` rows (one per cached device or 2 placeholders) until the fetch resolves. Same pattern for the usage chart wrapper.

4. **Localize the hardcoded `'ERROR'` toast** at `index.html:2965`:
   - Add `error` key to both `I18N.fa` ("خطا") and `I18N.en` ("Error").
   - Replace `toast('ERROR')` with `toast(I18N[LANG].error)`.

5. **`:focus-visible` rings** on the canonical copy + overflow buttons:
   ```css
   button:focus-visible {
     outline: 2px solid var(--accent);
     outline-offset: 2px;
   }
   ```

## Phase 6 — Motion gating + a11y polish

**Goal**: Honor `prefers-reduced-motion` system-wide; pass WCAG AA on every primary text token.

**What to implement**:

1. **Audit every remaining `transition:`, `animation:`, `@keyframes`**:
   ```bash
   grep -nE 'transition:|animation:|@keyframes' /root/subforme/index.html
   ```
   For each result NOT already inside a `@media (prefers-reduced-motion: no-preference)` block, wrap it. The blanket `*{}` rule from Phase 1 is the safety net; this phase makes it intentional.

2. **WCAG AA contrast pass**. The audit flagged `--ink-3 #6a6a78` on `--surface rgba(255,255,255,0.04)` = 2.2:1. Light theme value above (`--ink-3: #52525b` on `--surface: #ffffff`) calculates to 7.5:1 ✓. Dark theme value (`--ink-3: #9b9bab` on `--surface: #14141c`) calculates to 6.4:1 ✓. Verify with a contrast tool; adjust if off.

3. **Inspect every place that uses `--ink-3`/`--ink-mute`** as a text color and confirm the contrast still passes against its actual background (some text sits on `--surface-2`, which is lighter than `--surface` in light mode — recompute).

4. **Keyboard reachability**: every interactive control must be reachable via Tab in DOM order. Run through:
   - Topbar: lang button, theme button.
   - Hero: (no controls in the new layout if Phase 3 removed avatar/tier; just the copy button).
   - Devices: refresh button + retry button when in error state.
   - Configs: per-row Copy + ⋯, repeating.
   - `<details>`: summary is a button; inside, OS rail + apps + support.
   - Modals + action sheet: focus is trapped via `<dialog>`'s native behavior.

**Verification checklist for Phase 6**:
- [ ] `grep -nE 'animation:|transition:' index.html | grep -v 'reduced-motion'` returns a controlled set — every result is either inside a `@media (prefers-reduced-motion: no-preference)` block OR a transition the team explicitly decided is essential (the audit's view: there are zero essential idle animations on this page).
- [ ] Manual: enable OS reduced-motion → reload → no animation anywhere on idle.
- [ ] WCAG AA pass for every text token in both themes. Use a contrast checker; record the numbers in the commit message.

**Anti-pattern guards**:
- Don't add `transition: opacity 0.2s` "for polish" outside the prefers-reduced-motion: no-preference gate.
